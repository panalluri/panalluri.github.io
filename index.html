About Me

I am PhD student in the Applied Mathematics at Columbia University. My primary interest is in using machine learning to improve computational models of oceanographic systems. My research is conducted at Lamont-Doherty Earth Observatory and LEAP (Learning the Earth with Artificial Intelligence and Physics), a NSF STC focused on using machine learning to improve near-term cliamte predictions. I am also an affiliate with M2LInES (Multiscale Machine Learning In Coupled Earth System Modeling), an international project that uses machine learning to capture unaccounted physical processes at the air-sea-ice interface in order to improve climate projections. Currently, my work focuses on using neural networks to parametrize the effect of small-scale air-sea fluxes on large-scale atmosphere and ocean dynamics.

  • Parameterizing sub-mesoscale tracer transport

    Idealized modeling of sub-mesoscale tracer transport to develop machine learning based parametrizations and investigate novel data analysis methods. Undergraduate Research

My undergraduate research centered around Computational Fluid Dynamics (CFD) and mathematical physics, with a strong applied math foundation.

  • How does snow age? Quantifying coarsening of dry snow microstructure through phase-field simulations

    Computationally modeled dry snow metamorphosis by developing codes for and utilizing nonlinear PDE solvers. Created MATLAB image analysis scripts to quantify snow particle geometry evolution. Developed scripts to analyze particle coarsening rate at various temperatures.

    Slides from talk

  • CFD visualization in Trixi.jl

    Developing more efficient numerical methods to computationally model fluid flows. Improving on codes to visualize Julia outputs from fluid flow problems, using packages such as Trixi.jl. Computationally developing and verifying numerical methods of quasi-1-D flow problems.

    GitHub

  • A methodology for porting CUTEst optimization problems to Julia

    Developed native Julia ports of test problems from CUTEst, a collection of test problems used to hone mathematical optimizers. CUTEst test problems are mostly only available in Julia via an interface that generates Fortran code from a problem description, then compiles and invokes that Fortran implementation. I developed a Julia module that supports capabilities that the older CUTEst infrastructure supports; added further functionalities, such as computing in arbitrary precision.
